# 智能算法学习助手项目构思说明

## 1. 我对作业要求的理解

这次作业的重点不是简单接入一个大模型做问答，而是围绕我自己的真实学习特点，设计并实现一个能够持续辅助学习的个性化学习助手。它需要体现出明确的学习问题建模、AI 参与机制、完整的学习闭环，以及针对错误的高质量反馈。

结合评分标准，我理解这份作业最关键的要求有以下几点：

1. 需要先明确我在算法学习中的真实瓶颈，而不是泛泛地说“辅助我学习算法”。
2. 需要说明 AI 在系统中扮演什么角色，以及在什么情况下介入，而不是只有单轮问答。
3. 需要形成完整链路：输入题目或知识点、AI 分析、我进行尝试、AI 给出反馈、再进行强化或迁移练习。
4. 需要让系统体现出个性化特点，也就是这个助手是为我的学习方式服务的，而不是一个通用聊天机器人。
5. 需要有实际可运行的实现，至少关键逻辑和主要功能要真实落地。

因此，这个项目要尽量避免做成“题目一贴，AI 直接给答案”的工具，而应做成“帮助我识别题型、理解为什么选择某种算法、调用自己的模板并完成学习闭环”的学习系统。

## 2. 我对你想法的理解

我理解你的核心想法是做一个以个人模板库为基础的算法学习助手。这个助手的基础能力不是通用搜索，而是围绕你自己平时积累的算法模板展开，尽量贴合你的学习习惯和做题路径。

你的想法中至少包含以下几个核心目标：

1. 把你现有的所有算法模板统一整理进系统，作为你个人专属的知识库。
2. 当你遇到一道题目时，系统能够先分析题目特征，判断它更可能属于哪一类算法或数据结构。
3. 在完成算法判断后，系统能够从你的模板库中调出最匹配的模板，帮助你快速定位学习材料。
4. 系统不仅是“查模板”，还要帮助你学习，也就是让你理解为什么是这个算法、这个模板应该怎样用、和题目之间应该如何对应。
5. 你希望接入 DeepSeek 或其他性价比较高的大模型，让它承担题意分析、算法匹配、解释生成等工作。

换句话说，你真正想做的不是一个模板仓库，而是一个“题目分析 + 模板调用 + 学习引导”的个人算法学习助手。

## 3. 我打算怎么落实你的想法

我打算把这个项目拆成三个层次去落实，这样既贴合作业要求，也便于一步步实现。

### 第一层：把模板库变成可检索、可理解的知识库

你现在已经有一个分类较清晰的模板库，但目前主要还是按文件夹和代码文件组织。为了让 AI 稳定调用这些模板，我会先把模板库从“代码集合”整理成“带标签的知识条目”。

每个模板后续可以补充这些信息：

1. 模板名称
2. 所属类别，比如图论、二分、背包、线段树
3. 适用题型特征
4. 识别关键词
5. 前置知识
6. 时间复杂度和空间复杂度
7. 常见误用点
8. 模板代码路径
9. 示例题和使用说明

这样做的目的，是让系统以后不仅能“找到代码”，还能解释“为什么推荐这个模板”。

### 第二层：加入题目分析与模板匹配能力

当用户输入一道题后，系统不直接返回代码，而是先完成以下流程：

1. 抽取题目中的关键信息，比如数据范围、图结构、最短路、区间操作、背包容量、状态转移等信号。
2. 根据这些信号给出候选算法或数据结构，并说明判断依据。
3. 从你的模板库中检索最匹配的模板，并给出推荐理由。
4. 展示这个模板和当前题目之间的对应关系，例如哪些变量要改、哪些地方是通用部分、哪些地方需要按题意改造。

这一部分可以由规则检索和大模型推理结合完成。规则负责提高稳定性，大模型负责做解释和归纳，这样比单纯依赖模型更稳，也更适合课程作业展示。

### 第三层：把系统做成学习助手，而不是模板调用器

为了符合课程要求，系统还需要体现学习闭环。我打算加入以下设计：

1. 在调出模板前，先让你自己回答“这题像哪一类问题”或“你准备怎么做”，再由 AI 判断你的思路是否正确。
2. 如果你判断错了，AI 不直接给完整答案，而是指出你是错在题型识别、方法选择、状态设计、复杂度判断还是实现细节。
3. 在讲解完当前题目后，系统再生成一个相似但稍有变化的小题，帮助你验证是否真正掌握。
4. 系统记录你在哪些算法类别上容易反复出错，后续优先推荐对应模板和练习，形成个性化学习路径。

这样一来，这个助手的核心价值就不只是“帮我找模板”，而是“帮我建立从题目特征到算法选择，再到模板迁移和复盘反思的学习过程”。

## 4. 现阶段可执行的落地顺序

为了让项目更容易推进，我建议先按下面顺序做：

1. 先整理模板库，建立模板元数据。
2. 再实现题目输入后推荐模板的基础流程。
3. 然后补上推荐理由和学习引导逻辑。
4. 最后加入错因记录、复盘和强化练习。

如果后续继续推进，我会优先把这个项目整理成一版正式的设计方案，包括功能模块、数据结构、页面流程和技术实现路线。
