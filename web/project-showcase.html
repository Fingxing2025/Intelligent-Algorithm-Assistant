🎯 问题背景

算法学习者的真实困境

作为一个算法竞赛学习者，模板积累是长时间的过程。但当模板越来越多，真正做题时却常常卡在第一步——这道题到底该用哪个模板？

📂

模板散乱，查找低效

几十个 cpp 文件分散在本地多个文件夹里，每次翻找都靠记忆和运气，没有统一的检索入口。

❓

文件名看不出"怎么用"

光看 dijkstra.cpp 无法知道它适合什么题、有哪些坑、和当前题目有什么关系。

🧩

题 → 模板 匹配靠直觉

面对新题，常常停留在"这题好像见过"的阶段，缺少按题意快速匹配模板的工具。

🔄

积累越多，维护越难

每学一个新算法就要手动分类、写注释、记笔记，久了就容易滞后甚至遗忘。

🔄 方案对比

从"代码仓库"到"学习系统"

❌ 传统方式

  • • 手动翻文件夹找模板
  • • 文件名和代码分离，无标签
  • • 做题时凭感觉选算法
  • • 新模板要手工分类存放
  • • 复习全靠自己回忆

✅ 本系统

  • • 分类树 + 关键词搜索，秒定位
  • • 每个模板带适用题型/识别信号/风险
  • • 上传题目图片，AI 推荐候选模板
  • • 粘贴代码 → AI 分类 → 一键入库
  • • 收藏 + 最近查看，复习有迹可循
