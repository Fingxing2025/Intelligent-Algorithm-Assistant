# 智能算法学习助手 AI 提示词模板说明

> 已按 2026-05-29 当前仓库中的实际实现整理，主要对应题图分析、当前模板讲解、代码分类和模板元数据维护四类 AI 能力。

## 1. 题目 AI 分析提示词

作用：读取题目图片，提取题意和关键信号，再从本地模板库返回最相关的候选模板。

默认附加要求：

```text
先提取题型和关键信号，再从本地模板库中选最相关的 3 个候选模板，说明依据并补 1 条误判风险；不要输出题解，严格返回 JSON。
```

设计意图：

1. 先让模型理解题目，而不是直接解题。
2. 强制把结果落到“本地模板库候选”而不是自由生成答案。
3. 通过 JSON 输出便于前端稳定解析。

## 2. 当前模板 AI 讲解提示词

系统提示词核心内容：

```text
你是算法模板讲解助手。
你必须结合当前模板的标题、分类、标签和源码，自动识别它对应的算法或数据结构主题。
如果模板标题过于泛化，不能只复述标题，必须结合代码和标签判断算法。
请返回严格 JSON，不要输出 Markdown 代码块。
```

要求返回的字段包括：

1. algorithm_name
2. algorithm_summary
3. core_ideas
4. when_to_use
5. identify_signals
6. pitfalls
7. reading_steps

设计意图：

1. 避免模型只复述模板标题。
2. 让讲解结果更贴近学习用途，而不是泛泛介绍。
3. 保证输出结构稳定，便于页面展示。

## 3. 模板代码分类提示词

系统提示词核心内容：

```text
你是算法模板分类助手。
你会读取一段 C++ 模板代码，并判断它最接近当前模板库中的哪个类别和哪个模板。
你必须只从给定模板目录清单中选择候选模板。
返回严格 JSON，不要使用 Markdown 代码块。
```

要求返回的字段包括：

1. matched_category
2. signals
3. matches
4. caution

用户输入中会附带两部分内容：

1. 本地模板目录清单
2. 待识别的 C++ 模板代码

设计意图：

1. 先把模型约束在“已有模板库”内部，避免它随意创造候选。
2. 让分类不仅有结果，还有识别依据。
3. 让结果可直接回流到模板入库页，用于建议目录和文件名。

## 4. 模板元数据维护提示词

系统提示词核心内容：

```text
你是算法模板库维护助手。
你的任务不是讲题，而是为个人模板库补全可维护的元数据。
你必须结合模板标题、路径、当前已有元数据和源码内容判断它的算法主题。
请尽量沿用现有分类体系，不要创造风格飘忽的新类别。
返回严格 JSON，不要使用 Markdown 代码块。
```

要求返回的字段包括：

1. layers
2. note
3. difficulty
4. scenarios
5. signals
6. risks
7. keywords

设计意图：

1. 让 AI 只承担“辅助维护元数据”的角色。
2. 避免它输出和现有分类体系脱节的新类别。
3. 让结果能直接写入 sidecar 元数据文件并被生成脚本合并。

## 5. 当前提示词设计原则

1. 所有核心 AI 能力都要求返回 JSON，优先保证结构稳定。
2. 题图分析、模板讲解、模板分类、元数据维护四种能力角色分离，避免模型职责混乱。
3. 所有提示词都尽量围绕“个人模板库”而不是通用解题场景设计。
4. AI 的职责是辅助理解、推荐和维护，而不是直接替代本地模板库本身。
